# Build and verification

## Toolchain

- Windows x64
- Go 1.26.1 (pinned by `go.mod`)
- GNU `windres` available on `PATH`
- Windows PowerShell 5.1 or newer PowerShell

The project currently has no third-party Go modules. Build caches and temporary files are redirected to `.cache/go-build` and `.tmp/go` inside the project.

## Commands

Run the complete source checks:

```powershell
./scripts/test.ps1
```

The test script runs all normal vet analyzers except `unsafeptr`. Win32 callbacks must reconstruct SDK-owned structures from `LPARAM`; those conversions are isolated to the shell callback boundary and are covered by the Windows integration tests.

Build console Debug and Windows GUI Release executables:

```powershell
./scripts/build.ps1
```

Verify the real emitted PE subsystem, icon, VERSIONINFO and portable layout:

```powershell
./scripts/verify-artifact.ps1
```

Run the S02 GUI lifecycle, second-instance, tray, recovery and stress checks:

```powershell
./scripts/test-s02-shell.ps1
```

Run the S03 input state-machine, hook lifecycle and swallowed real `SendInput` capture gates:

```powershell
./scripts/test-s03-input.ps1
```

Optionally run a one-minute captured soak for each of keyboard/left/right (about three minutes total):

```powershell
./scripts/test-s03-input.ps1 -SoakMinutes 1
```

Clean generated artifacts:

```powershell
./scripts/clean.ps1
```

## Versioning

- `VERSION` is the single source for the product SemVer.
- `scripts/build.ps1` stamps the same version into Go build information and Windows VERSIONINFO.
- The numeric Windows file version adds a fourth zero component; `0.1.0` becomes `0.1.0.0`.
- The Git commit, UTC build time, Go version and target are recorded in `dist/build-info.json`.
- Set `BUILD_COMMIT` to override the commit identity in source archives without `.git`.
- Set `SOURCE_DATE_EPOCH`, or pass `-BuildTimeUtc`, to make the recorded build time deterministic.

Example deterministic invocation:

```powershell
$env:SOURCE_DATE_EPOCH = '1784260800'
$env:BUILD_COMMIT = '0123456789ab'
./scripts/build.ps1 -Configuration Both
```

## Generated files

- `assets/app.ico` is deterministically generated by `tools/icon`; it is not a downloaded asset.
- `build/app.generated.rc` and `cmd/genshin-tools/app.syso` are generated by the official build script.
- `dist/GenshinTools-debug.exe` uses the console subsystem for diagnostics.
- `dist/GenshinTools.exe` uses the Windows GUI subsystem for release.
- Runtime state belongs under `data/` next to the executable, with `logs/`, `cache/` and `staging/` below it.
