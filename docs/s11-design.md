# S11 程序设置、语言与 UI 收尾设计

制定日期：2026-07-18  
上游审计基线：`b5a050ebd319341bddc4189491c90c22162d33fa`

## 1. 上游设置审计与取舍

上游设置页混合了主题、语言、窗口、托盘、优先级、截图、更新、背景图片/视频、签到账号、安全认证和大量首页卡片设置。本项目只保留与轻量原生启动器直接相关的部分：

| 上游能力 | 本项目处理 |
|---|---|
| 主题、语言、托盘、窗口尺寸、最小尺寸、进程优先级 | 保留并用 Go/Win32 重写 |
| 版本、许可证、诊断信息 | 保留并强化为可导出的脱敏报告 |
| 本程序更新 | 延后到 S12，S11 只展示当前版本和上游审计 SHA |
| 首页新闻/签到/每日便笺/抽卡卡片 | 永久排除，不创建配置键或后台服务 |
| 安全认证、账号、Cookie、登录 | 永久排除 |
| 在线/视频背景、轮播、亚克力和复杂皮肤 | 不采用；保持简单、轻量暗色界面 |
| 高/实时进程优先级 | 不采用；只允许 BelowNormal/Normal/AboveNormal，避免饿死系统或输入线程 |

## 2. 配置模型

配置 schema 升级，新增独立 `shell` 设置：

- `language`：`system`、`zh-CN`、`en-US`；system 只在简中系统选择中文，其余选择英文。
- `theme`：`dark` 或 `system`；system 跟随系统明暗，高对比度始终由系统颜色覆盖。
- `minimizeToTray`：决定最小化时隐藏到托盘还是保留任务栏按钮。
- `rememberWindowSize`：关闭时是否保存位置和尺寸。
- `enforceMinimumSize`：是否启用安全最小客户区；关闭后页面使用自适应压缩布局。
- `processPriority`：`below-normal`、`normal`、`above-normal`。
- `cpuWarningEnabled` 与有界阈值；只提示本程序持续异常占用，不做强制内存清理或自杀式重启。

旧 schema 迁移使用安全默认值，不自动改变用户当前托盘行为和窗口尺寸。

## 3. 本地化结构

新增 `internal/localization`：

- 稳定资源 key 与简中/英文内置表；缺失 key 回退英文，再回退 key 本身。
- 资源表在测试中强制 key 集完全一致，禁止某语言静默缺文案。
- shell 只通过资源 key 获取可见文本；日志事件名、协议字段和持久化枚举不翻译。
- 切换语言立即刷新导航、页面、托盘提示和错误状态；不会重启后台任务。

当前 shell 约有 300 条中文可见字符串，迁移分两步完成：先建立资源基础和设置/导航页面，再逐页替换所有保留功能文本。永久排除功能字符串必须保持为零。

## 4. 主题与高对比度

- 颜色集中到 `Palette`，页面不再散落硬编码刷色。
- `dark` 使用本项目简洁暗色；`system` 读取 Windows 应用主题。
- 检测 `SPI_GETHIGHCONTRAST` 后强制使用 `COLOR_WINDOW`、`COLOR_WINDOWTEXT`、`COLOR_HIGHLIGHT` 等系统色，不自行模拟高对比度。
- 主题变化只重建有限画刷并 Invalidate；旧 GDI 对象必须释放，不能随切换泄漏。
- EDIT 控件、标题栏、托盘和覆盖层分别验证，不添加亚克力、背景媒体或持续动画。

## 5. 设置、关于与诊断页面

设置页使用紧凑分区，并共享现有 DPI/自适应坐标：

1. 语言与主题。
2. 托盘、窗口记忆和最小尺寸。
3. 进程优先级和 CPU 异常提醒。
4. 关于：本项目版本、构建提交、构建时间、Go 版本、上游审计 SHA、范围策略。
5. 诊断导出：用户选择目录后写入 UTF-8 JSON，包含脱敏后的构建/OS/显示器/DPI/资源计数/配置摘要和近期日志；不包含账号、Cookie、Token、完整游戏路径或插件目录源查询文本。

帮助文档仍属于永久排除项；“关于”和诊断不是帮助浏览器，也不会打开网页。

## 6. Win32 稳定性门禁

- UI 线程不读取大日志、不枚举大目录；诊断采集和导出走后台任务。
- 设置写入继续使用 schema 校验与原子替换；应用失败时恢复内存值并显示错误。
- 进程优先级调用失败不影响启动和输入增强。
- 最小化行为必须区分程序设置、游戏启动后行为和托盘恢复，不形成隐藏后无法找回的窗口。
- DPI 变化、主题变化和语言变化后，子控件字体/位置/可见性同步更新。
- 键盘上下导航、Tab 焦点和 Enter/Space 激活不能启动重复后台任务。
- 诊断导出路径限定为用户显式选择的目标；临时文件写入、Sync 后原子提交。

## 7. S11 退出门禁

1. 简中与英文资源 key 完整一致，所有保留页面无硬编码单语可见文本。
2. 100%/125%/150%/200% DPI、最小窗口和多显示器边界自动化/人工矩阵有记录。
3. dark/system/high-contrast 三种路径可读，反复切换不增长 GDI/USER 对象。
4. 托盘、窗口、优先级和 CPU 提示设置可迁移、持久化和失败回退。
5. 关于信息可追踪；诊断导出脱敏且不阻塞 UI。
6. 永久排除功能没有导航、配置键、字符串、网络初始化或隐藏入口。
7. `0.10.0` 的测试、竞态、短压力、正式构建和 VERSIONINFO 通过。
