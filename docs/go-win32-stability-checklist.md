# Go + Win32 稳定性风险与验收清单

本清单是每个 Phase 的构建门禁，不是上线前才看的附录。编号应进入 issue、测试用例或提交说明；只有提供证据或写明不适用理由才能关闭。清单覆盖当前可预见的主要崩溃、卡死和资源失控类型，但不存在能数学证明“永不出现新类型”的静态列表；项目中发现的新模式必须先补入清单，再修复和回归。

严重级别：

- P0：可能崩溃、卡死、持续注入输入、损坏游戏文件或无法退出。
- P1：功能失效、资源持续泄漏、状态错误、只能强杀恢复。
- P2：性能下降、短暂假死、显示或诊断问题。

## A. Go/Win32 ABI 与 unsafe

| ID | 风险/典型表现 | 预防与验证 | 级别 |
|---|---|---|---:|
| A01 | WinAPI 参数类型、返回值或 calling convention 声明错误，调用即崩 | API 封装集中；对照 SDK；x64 集成测试 | P0 |
| A02 | `int`/`LONG`/`WPARAM`/`LPARAM`/指针宽度混用，句柄截断 | 固定宽度结构；禁止把句柄存入 32 位字段 | P0 |
| A03 | 结构体对齐、union、`cbSize` 错误导致内存越界 | 编译期 size/offset 断言；与 C 布局对照 | P0 |
| A04 | UTF-16 缓冲区无终止符、容量按字节/字符算错 | 统一 UTF-16 helper；边界与长文本测试 | P0 |
| A05 | 含嵌入 NUL 的 filter/多字符串被普通转换函数截断或 panic | 专用 double-NUL 构造器和测试 | P0 |
| A06 | Go 指针被 Win32 异步保存，GC 移动/回收后访问 | 异步上下文放稳定内存；明确所有权；`runtime.KeepAlive` | P0 |
| A07 | syscall callback 被 GC 回收或函数返回后仍调用 | callback 与窗口/服务同生命周期持有 | P0 |
| A08 | WndProc/hook callback panic 穿过系统回调边界 | 最外层 `recover`、崩溃日志、返回安全默认值 | P0 |
| A09 | `GetLastError` 在其他调用后才读取，错误被覆盖 | 调用封装立即捕获 errno | P1 |
| A10 | 使用无效/已关闭 HANDLE、重复 `CloseHandle` | 单一所有者、幂等 Close、句柄跟踪 | P0 |
| A11 | DLL Proc 缺失却使用 MustFind 或延迟到运行时突然 panic | 启动能力探测；可选 API 有 fallback | P0 |
| A12 | x86/x64/ARM64 架构不匹配 | 首版只发 x64；启动时检查 PE machine | P0 |

## M. 阶段门禁与运行时观察

每个 Phase 至少记录：

- commit、构建命令、Go 版本、Windows 版本、权限级别。
- 用例开始/结束时：goroutine、线程、HANDLE、USER、GDI、内存、CPU。
- UI watchdog 最大延迟；输入 hook 回调 P50/P95/P99 延迟（输入阶段）。
- 所有后台任务取消至停止的最长时间。
- 异常退出后重启能否识别并进入安全模式。
- 仍开放的 P0/P1 编号；存在开放 P0 时禁止生成正式发布包。

建议的最低压力场景：

1. 主窗口打开/关闭和托盘恢复各 1000 次。
2. 导航切换 5000 次，控件和 GDI 数不持续增长。
3. 输入启停 200 次、边沿事件 1000 次、三模式短节拍矩阵；长时间运行按发布风险在 S13 选择执行。
4. 下载取消/重试 100 次，断网、限速、磁盘满和锁文件各覆盖。
5. 游戏快速启动退出 100 次；程序先退、游戏先退两种顺序。
6. 睡眠/唤醒、锁屏/解锁、RDP 连接变化、显示器拔插、DPI 切换。
7. 配置截断、乱码、旧版本、只读、并发写和迁移中断。
8. 注入模块缺失、损坏、错架构、错版本、helper 超时；纯净启动必须仍可用。
